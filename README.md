# srnmSchoolBot

Telegram-бот для ведения школьного расписания, домашних заданий и уведомлений о предстоящих уроках в классном чате.

Бот работает на `aiogram 2.x`, хранит данные в MySQL и планирует уведомления через `APScheduler`.

## Возможности

- создание и настройка классов;
- привязка класса к Telegram-группе через `/verify`;
- добавление администраторов класса;
- создание групп внутри класса;
- ведение расписания по дням;
- добавление уроков с названием, временем, длительностью, домашним заданием и
  местом проведения;
- поддержка еженедельных уроков;
- создание одного урока сразу для всех групп класса;
- уведомления перед уроком;
- ежедневные уведомления о расписании на завтра;
- выбор языка интерфейса: `en`, `ru`.

## Требования

- Python 3.9+;
- MySQL;
- Telegram Bot Token от [@BotFather](https://t.me/BotFather).

## Установка

```bash
git clone <repo-url>
cd srnmSchoolBot

python -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

## Настройка

Создайте файл `settings.py` на основе примера:

```bash
cp settings_reference.py settings.py
```

Заполните значения:

```python
TOKEN = 'telegram-bot-token'
ADMINS = [123456789]

DB = {
    'host': 'localhost',
    'user': 'user',
    'password': 'password',
    'name': 'database_name',
}

LANGS = ['en', 'ru']
```

Где:

- `TOKEN` - токен Telegram-бота;
- `ADMINS` - Telegram ID администраторов, которым бот отправляет сообщения о
  запуске и ошибках;
- `DB` - параметры подключения к MySQL;
- `LANGS` - доступные языки интерфейса.

Перед запуском создайте базу данных MySQL, указанную в `DB["name"]`. Таблицы
создаются автоматически при первом старте, если подключение к существующей
структуре не удалось.

## Запуск

```bash
python main.py
```

После успешного запуска бот отправит администраторам сообщение:

```text
Bot is started!
```

## Основные команды

- `/start` - регистрация пользователя и приветствие;
- `/help` - справка по работе с ботом;
- `/support` - контакты поддержки;
- `/settings` - настройки пользователя;
- `/class` - управление классами, группами и расписанием;
- `/verify` - привязка класса к Telegram-группе. Команду нужно отправить в
  групповом чате от имени администратора чата.

## Типовой сценарий

1. Напишите боту `/start` в личных сообщениях.
2. Откройте `/settings` и выберите язык.
3. Откройте `/class` и создайте класс.
4. Добавьте бота в Telegram-группу класса.
5. В группе отправьте `/verify` от имени администратора.
6. Вернитесь в личные сообщения с ботом, откройте `/class` и настройте группы,
   расписание, часовой пояс и уведомления.
7. В групповом чате используйте `/class`, чтобы смотреть расписание.

## Структура проекта

```text
.
├── main.py                  # точка входа, запуск polling
├── handlers.py              # обработчики команд, сообщений и callback-запросов
├── keyboards.py             # inline-клавиатуры
├── langs.py                 # тексты интерфейса на разных языках
├── models.py                # FSM-состояния aiogram
├── utils.py                 # вспомогательные функции и планирование уведомлений
├── db/
│   ├── main.py              # подключение к MySQL и создание таблиц
│   └── model.py             # ORM-модели
├── docs/                    # схемы структуры бота и базы данных
├── settings_reference.py    # пример локальной конфигурации
└── requirements.txt         # зависимости Python
```

## Документация

В папке `docs/` лежат дополнительные материалы:

- `bot_structure.pdf` и `bot_structure.xmind` - структура сценариев бота;
- `db_structure.pdf` и `db_structure (via dbdiagram.io).txt` - схема базы
  данных.

## Примечания

- `settings.py` содержит секреты и не должен попадать в Git.
- Бот использует `MarkdownV2`, поэтому пользовательский текст экранируется в
  `utils.py`.
- FSM-состояния хранятся в памяти (`MemoryStorage`), поэтому при перезапуске
  незавершенные пользовательские сценарии сбрасываются.
